Introduction to CSS

CSS, Cascading Style Sheets, serve to customise elements in your HTML project. The syntax is the following: Selector, followed by curly braces, which take in a property: value input. The selector, chooses a HTML element. The property chooses a specific property of the HTML value, and the value manipulates the value of the HTML property.

The Universal Selector

The universal selector selects all elements, and the syntax for it is *.

The Type Selector

The type selector, or otherwise called the element selector, selects a particular element. The syntax for this is just the name of the element.

The Class Selector

The class selector selects a particular class. The class selector syntax is a period followed by the exact name of the class. A note about adding a class attribute to a specific HTML element is that classes are made by a particular string of text, and adding a space indicates multiple classes, so hyphens are used when naming classes with more than a word. For a clearer explanation, please clik here

The ID Class Selector

The ID class selector selects the id, since any id is unique. The ID selector syntax is a hash symbol, and immediately after the exxact string combination which forms its name. It is said that a common pitfall is using ID selectors, when class selectors will suffice. ID selectors should be used sparingly, if at all.

Chaining Selectors

It is a common occurence when you would like to select multiple elements, classes or id's. Sometimes you would like to select each together within a single css command.

Multiple Selectors

To chain multiple classes, we could either list them out using a comma as a seperation, or we could have them together, without either a comma or a space. The same can be used with id's, where you place the syntax immediately after the end of the name of a class. You are not able to chain multiple elements.

Combinators

Sometimes you would like to select classes inside other classes. From here, using the Descendant Combinators method will allow you to do as such. The syntax for this operation is to add a space between the two classes/id's/elements, and followed by the regular notation. For using the descendant combinator method on two classes for example, we would: .class1.class2.

Order Matters

In the cases where there is a style that applies a property twice. The style that was applied last, will be the one that will appear in your document.

Place a href attribute on the link, and set it equal to the location of the css file. Place a rel attribute, and type stylesheet. This shows the relationship between the html doccument and the css file, where the css file acts as the html file's stylesheet.

CSS Cascading

Specificity

CSS places significance over which styles appear. In the case where there are conflicting styles, CSS will decide to present the stlye which is the most specific.

  • ID selector
  • Class selector
  • Type selector
  • Universal selector

An ID selector, will always beat over any amount of class selectors. A class selector will always beat over any amount of type selectors. And lastly, type selectors, which always come last in the specificity chain.

Suppose there is a style which uses a kind of selector, and there is another style, which uses the same kind of selector, but twice. The style which uses the same selector twice, is the style which takes precedance and takes appearance in the final document.

Inheritance

Cases of inheritence refer to when selectors are nested within each other. Styles which target through an inheritence take less precedancce over styles which target directly.
